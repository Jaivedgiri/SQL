E-commerce Sales Analysis using SQL
Project Overview

Conducted data analysis on an E-commerce sales dataset using SQL to evaluate business performance, customer purchasing behavior, and product-level trends. The project focused on solving real-world business problems through structured querying and data-driven insights.

Key Responsibilities
Structured and analyzed relational data across multiple tables including Customers, Orders, Order_Items, Products, and Payments
Performed data cleaning by handling missing values, duplicates, and inconsistencies
Applied SQL Joins (INNER, LEFT) to integrate data from multiple sources
Utilized aggregate functions (SUM, COUNT, AVG) to calculate key business metrics
Implemented GROUP BY for category-wise and region-wise analysis
Used Window Functions (RANK, ROW_NUMBER) to identify top-performing products and customers
Developed CTEs and Subqueries to handle complex analytical queries
Optimized SQL queries to improve performance and readability
Business Problems Solved
Identified top-selling products and highest revenue-generating categories
Analyzed customer purchase behavior and high-value customers
Evaluated monthly and yearly sales trends
Assessed region-wise sales performance
Calculated key revenue and order distribution metrics
Key Metrics (KPIs)
Total Revenue
Total Orders
Average Order Value (AOV)
Customer Contribution to Revenue
Product and Category Performance
Key Insights / Impact
Identified products contributing significantly to overall revenue, enabling better inventory planning
Highlighted high-value customers to support targeted marketing strategies
Revealed sales trends and seasonality for improved business forecasting
Provided actionable insights to support data-driven decision-making
